5. Implementation decisions

Not sticky. The Design System never defines sticky behavior, and the brief said sticky "only if defined in the implementation guide." So the header scrolls with the page.

No footer icons. I initially used Lucide Github/Linkedin/Mail, but lucide-react 1.24 has dropped brand icons — only generic Git* glyphs remain. Adding a brand-icon package would violate the dependency rules, and section 26 says icons must never replace text, so the labels "GitHub", "LinkedIn", "Email" stand alone. Lucide is still used for the menu open/close icons, where the affordance is genuinely iconographic.

Panel stays mounted. invisible (not hidden) keeps both directions of the transition working while still removing closed links from the tab order.

Menu closes at the md breakpoint. If the viewport crosses 48rem while the menu is open, the container's md:hidden would hide the panel with body still scroll-locked and no way to unlock it. A matchMedia listener closes the menu instead.

Route changes close the menu via render-phase state adjustment, not an effect — react-hooks/set-state-in-effect rejects the effect form, and this also covers browser back/forward, which a link onClick cannot see.

Footer keeps its navigation list. The brief listed only copyright/GitHub/LinkedIn/Email, but 04_DESIGN_SYSTEM (§ footer) requires primary navigation links in the footer, and the higher-priority document wins.

ExternalProfile.href is "#" — placeholders only, per the no-real-content constraint.
