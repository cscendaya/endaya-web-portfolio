# Project Overview

This project is a personal web portfolio developed as part of the Bachelor of Science in Information Technology program.

The initial version is created primarily to satisfy the academic portfolio requirement while establishing a professional online presence.

Although Version 1 is intentionally limited in scope, the project is designed with long-term maintainability in mind so it can evolve into a comprehensive professional portfolio after graduation.

The objective is not to build the most feature-rich portfolio possible, but to build one with a clear identity, solid engineering practices, and a scalable foundation.

# Project Principles

Every future decision should support the following principles.

## Simplicity

Avoid unnecessary complexity.

Every feature should have a clear purpose.

---

## Maintainability

Code should prioritize readability and organization over clever implementations.

Future expansion should be straightforward.

---

## Performance

Pages should load quickly.

Avoid unnecessary JavaScript.

Optimize assets where appropriate.

Performance is a design feature.

---

## Accessibility

Accessibility is a baseline requirement rather than an optional enhancement.

Interfaces should be usable with:

- keyboard navigation
- screen readers
- sufficient color contrast
- semantic HTML

---

## Consistency

Visual and technical consistency should take priority over experimentation.

Spacing, typography, components, and interaction patterns should remain predictable throughout the site.

---

## Authenticity

The portfolio should accurately represent current skills and experience.

Avoid exaggerated claims.

Do not imply expertise that has not yet been demonstrated.

---

# Design Philosophy

The visual language should communicate professionalism without unnecessary decoration.

Desired characteristics:

- Analytical
- Minimal
- Calm
- Modern
- Content-first
- Accessible
- Fast
- Intentional

The interface should remain visually quiet so that the content becomes the primary focus.

Animations should support usability rather than act as decoration.
